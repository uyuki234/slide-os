# slide-os

QEMU上で起動する、スライドを表示するためだけの最小OS風デモです。
Linuxのプロセスとしてではなく、仮想マシンの起動時に直接実行されます。

## できること

- Multiboot形式のカーネルとして起動する
- VGAテキストバッファ(0xB8000)への直接書き込みで文字を表示する
- PS/2キーボードをポーリングして、N/Pキーでスライドを送る

## できないこと(=普段OSがやってくれていること)

- プロセス管理 / メモリ管理 / ファイルシステム / ネットワーク
- 割り込み処理(キー入力はポーリングで代用)
- スキャンコード→文字の変換(キーの背番号を直接見ている)

## 必要なもの (macOS)

```sh
brew install qemu nasm i686-elf-gcc
```

## 実行方法

```sh
make run
```

## 構成

| ファイル  | 役割                                                 |
| --------- | ---------------------------------------------------- |
| boot.s    | 入口。Multibootヘッダ、スタック設定、kmain()呼び出し |
| kernel.c  | 本体。画面描画、スライドデータ、キー入力             |
| linker.ld | カーネルを1MB番地に配置する指示書                    |
| Makefile  | クロスコンパイルとQEMU起動                           |

## 起動の流れ

```text
qemu-system-i386 -kernel kernel.elf
  → QEMUがMultibootヘッダを確認し、1MBに配置(GRUB相当)
  → boot.s: スタックを用意して kmain() へ
  → kernel.c: 0xB8000に書く → 画面に文字が出る
```

## 学んだこと

- main() の前には、スタックすらない世界がある
- printfは「ない」のではなく、前提となるOSとlibcが存在しない
- 画面はメモリ書き込み、キーボードはポート入出力と、話し方が2系統ある
- AIには実装代行ではなく、1ファイルずつ説明させる使い方が学びになる
